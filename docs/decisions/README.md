# 결정기록 작성 규칙

public site 전용 기술 결정은 시간이 지난 뒤에도 화면과 구현이 왜 현재 형태가 되었는지 알 수 있도록 남긴다.

## 파일명

`NNNN-short-kebab-case-title.md` 형식을 사용한다.

예:

- `0001-fixed-site-navigation.md`
- `0002-financial-chart-axis-unit-format.md`

## 템플릿

```md
# NNNN 제목

Date: YYYY-MM-DD
Status: Proposed | Accepted | Superseded
Scope: public-site

## Context

왜 이 결정을 고민했는가.

## Decision

무엇을 선택했는가.

## Rationale

왜 이 선택이 타당한가.

## Consequences

좋아지는 점과 감수할 점.

## Related

관련 커밋, 파일, 루트 저장소 문서 링크.
```
