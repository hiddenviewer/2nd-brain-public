# 작업 에이전트 운영 규칙

이 저장소는 2nd-brain의 public GitHub Pages 사이트를 소유한다. Astro 페이지, 사이트 네비게이션, 회사 페이지, 재무 차트, 공개용 링크 목록, 정적 빌드 설정이 이 저장소의 책임이다.

## 저장소 역할

- public site에 보이는 화면과 정적 렌더링 로직을 관리한다.
- 회사 페이지, 카테고리 페이지, 링크 목록, 노트 페이지, 검색 인덱스, GitHub Pages 배포를 관리한다.
- ingest, Discord bot 명령, DART 조회 모듈, 위키 원천 데이터 구조는 루트 저장소의 책임이다.

## 결정기록과 실패기록

- public site 구현에만 해당하는 기술적 결정은 `docs/decisions/`에 기록한다.
- public site 구현에만 해당하는 실패와 재발 방지 지식은 `docs/failures/`에 기록한다.
- ingest, bot, DART, 위키 데이터, cross-repo 흐름에 걸친 결정은 루트 저장소의 `docs/`에 기록한다.
- 같은 내용을 루트 저장소와 public repo에 중복 기록하지 않는다. 필요한 경우 관련 문서 링크를 남긴다.
- 문서는 한글로 작성하되, 명령어, 파일명, API 이름, 코드 식별자는 원문을 유지한다.

## 검증 기준

- public site 코드나 데이터 표시 변경 후에는 `npm run build`를 실행한다.
- 네비게이션, 표, 차트, 회사 페이지 레이아웃을 바꾸면 데스크톱과 모바일 폭에서 표시가 깨지지 않는지 확인한다.
- 문서만 변경한 경우에는 빌드를 생략할 수 있다.

## 커밋 기준

- public site 변경은 이 저장소에서 별도 커밋한다.
- 루트 저장소 변경과 섞지 않는다.
- 커밋은 하나의 논리적 작업 단위로 나눈다.
- 배포가 필요한 변경은 push까지 수행한다.
